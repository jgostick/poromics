# Architecture Documentation

This folder documents the system architecture and operational patterns for Poromics beyond user-facing setup in the project `README.md`.

## Documents

- `system-overview.md`: high-level architecture, request flow, major components.
- `team-tenancy.md`: how team context and team-scoped data access work.
- `compute-routing.md`: Julia/Taichi queue and endpoint routing model.
- `operational-learnings.md`: practical lessons and gotchas from recent implementation work.

## Scope

These docs are implementation-oriented and intended for:

- Maintainers and contributors
- Coding agents and code-review sessions
- Architecture onboarding and troubleshooting

For local setup and app usage, continue to use the top-level `README.md`.
